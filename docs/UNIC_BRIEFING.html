The Agent Wall

Enterprises are deploying fleets of AI agents that read files, query databases, send emails, write code, and call each other. Every tool an agent can touch is attack surface. The most dangerous and most ignored direction is content flowing back from tools into the agent — a poisoned document, a manipulated API response, or a malicious tool description can silently reprogram a legitimate, fully-authorized agent mid-session. Perimeter security, firewalls, and human code review never see this traffic.

"Agents provide intelligence. Deterministic systems enforce authority."

The architectural law governing the entire system — an LLM may conclude "this looks suspicious"; only deterministic code decides "this session is quarantined."

The Full Security Cycle

Govern→ Detect→ Contain→ Reproduce→ Repair→ Verify→ Learn & Strengthen

The deeper philosophy is not only defense but adaptive resilience — the system should not merely survive an attack, it should understand it, repair the weakness, and become harder to attack next time. This is where Olive diverges from every competitor: everyone detects and blocks; almost nobody closes the loop.

Part 02 · Section C

Architecture

Olive is a transparent MCP proxy. To the agent's client it looks like a normal MCP server. To the real tool server it looks like a normal MCP client. Everything that crosses it is inspected in both directions. 

Agent ──► [ FAST PATH — deterministic, inline, ENFORCES ]──► Tool Server
                           │
                     telemetry (async)
                           │
              [ PARALLEL PATH — LLM sentinels, advisory only ]
                     signals ──► circuit breaker
                                 (deterministic enforcement only)
OliveGateway
gateway/proxy.py

The proxy. Builds a SecurityContext per call, runs the inspector pipeline in both directions, logs everything. Never trusts tool output.

SecurityContext
gateway/context.py

One frozen object per inspected message. Carries agent identity, role, tool, direction, session history, resource being accessed. What inspectors reason about — not raw content.

InspectorPipeline
gateway/pipeline.py

Runs inspectors in order. First non-allow verdict wins and short-circuits. Any inspector exception → block (fail closed, always). 80 lines.

CircuitBreaker
gateway/breaker.py

Owns all session state. When blocks exceed threshold → quarantine the entire session. The only object the intelligence layer can write to — via two narrow signals.

PolicyInspector
inspectors/policy.py

Outbound. Is this tool allowed for this role? Unknown tools are blocked by default — default deny, not default allow.

DecodeInspector + PatternInspector
inspectors/decode.py · patterns.py

Inbound. Decode layer strips base64/hex/rot13/homoglyphs before pattern matching. Injection phrases are caught regardless of encoding.

SentinelRunner
intelligence/sentinels.py

Runs LLM sentinels (PromptInjection, DataLeak, Behavior) on telemetry events asynchronously. Sole caller of breaker.trip() — one writer, one authority.

SecurityCommander
intelligence/commander.py

Deterministic, not an LLM. Sole caller of breaker.set_mode(). Escalates Normal → Suspicious → Siege from the incident stream. Only a human with olive:command token de-escalates.

IncidentBus
intelligence/bus.py

HMAC-signed async pub/sub between departments. Append-only audit log. Departments collaborate through structured objects — never uncontrolled group chat.

Four Runtime Departments
intelligence/departments.py · redteam_dept.py · builder_dept.py

Defense, Remediation, Red-Team, Builder. Each has a structural import-set guarantee it cannot touch live enforcement. Advisory and propose-only — humans gate every promotion.

RemediationLedger
intelligence/remediation.py

State machine: reproduced → fix-proposed → verified → approved → learned. Verify is writable only from the real eval gate result. Learn requires an olive:remediate token from a human.

Command Center (Web + TUI)
ui/web.py · ui/app.py · ui/static/

Read-only dashboard over WebSocket. Shows live incident stream, department status, and gate metrics. olive serve --ui runs gateway + dashboard in one process sharing one bus.

The Layering Rule (keeps the open-core split clean)

gateway core  ◄──── 2 narrow signals ────  intelligence/
              (breaker.trip · set_mode)
              gateway never imports outward

This seam is the potential open-core boundary: the gateway core is the open layer; the intelligence, fleet management, and enterprise control plane are the commercial layer. Both options stay alive because the split is always a clean cut.

Part 02 · Section E.5

Research Foundations

Olive's architecture is grounded in four established research areas — each applied directly to the MCP protocol. This is not a novel-technique story. It is a novel-application story: taking techniques that are well-understood in adjacent fields and being the first to apply them systematically to the agentic-tool layer.

Cryptographic Identity

SPIFFE / Workload Identity

SPIFFE (Secure Production Identity Framework for Everyone) defines the SVID standard for cryptographically provable workload identity — the same problem Olive solves for agents. JWT-SVID (X.509 + JWT hybrid) is the direct ancestor of Olive's capability-scoped RS256 tokens.

Applied as: per-agent RS256 JWT with embedded role, capabilities, task-resources, and session_id — issued by a mock CA today; plugs into a production SPIFFE-compatible CA without code changes.
Behavioral Security

Session Anomaly Detection

Academic work on HTTP session modeling (Kruegel & Vigna, 2003), behavioral baselines, and concept-drift detection underpins modern UEBA (User and Entity Behavior Analytics). The key insight: a single call can be innocent; the sequence reveals intent.

Applied as: cross-session behavioral baselines in the BehaviorSentinel (M10) — each agent's tool-call history is stored and compared across sessions to detect anomalous expansion of behavior over time.
Adversarial ML

Eval Corpus Methodology

Adversarial robustness evaluation (Carlini & Wagner, 2017; Goodfellow et al.) established the discipline of measuring defenses against actively optimized attacks, not just random noise. The red-team / eval corpus methodology in security follows the same principle.

Applied as: 107-case eval corpus with status:active / known-miss classification; CI-gated eval runner that blocks merges when detection degrades. Every bypass found by the red-team agent becomes a permanent regression case.
LLM Safety

Prompt Injection & Tool-Call Attacks

Research on indirect prompt injection (Greshake et al., 2023), tool-poisoning attacks, and rug-pull manipulation — directly catalogued in OWASP LLM Top 10 (LLM01, LLM02) — maps exactly to the MCP attack surface Olive defends.

Applied as: four inspector layers: policy (structural rules), pattern (regex + Unicode normalization), decode (base64/URL/hex bypass stripping), and LLM sentinel (advisory-only, never enforces). Covers all four OWASP categories in the corpus.
Olive is not proposing new cryptography or a new ML architecture. It is the first system to compose SPIFFE-grade identity, behavioral session analysis, adversarial eval methodology, and prompt-injection detection into a single transparent MCP proxy — making the resulting moat structural, not algorithmic.
