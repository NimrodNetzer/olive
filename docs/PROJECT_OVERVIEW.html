Vision

Every technological era creates a new attack surface, and therefore a new defensive layer. Swords created the need for shields. Networks created the need for firewalls. Autonomous agents create the need for the Agent Wall.

The central promise: let companies use autonomous agents without giving up control.

The most dangerous attack direction is the one almost everyone ignores: content flowing back from tools into the agent. A poisoned document, a manipulated API response, or a malicious tool description can silently re-program a legitimate, fully-authorized agent mid-session. Perimeter security, SAST, and human code review don't see this traffic at all.

What Olive protects against

  • External agents attempting to manipulate company agents
  • Legitimate agents compromised by prompt injection
  • Internal agents operating outside their assigned roles
  • Agents attempting to gain additional permissions
  • Agents extracting sensitive company information
  • One agent manipulating or impersonating another
  • Malicious content hidden inside files, websites, or API responses
  • Compromised MCP servers or tools (incl. tool-description "rug-pulls")
  • Several individually-harmless actions combining into one dangerous outcome
  • Agents making serious mistakes without malicious intent

The full security cycle

Govern → Detect → Contain → Reproduce → Repair → Verify → Learn & strengthen

Not just defense, but adaptive resilience: the system shouldn't merely survive an attack — it should understand it, repair the weakness, and become harder to attack next time. ("Olive" — olive trees regrow from the roots after being cut down or burned, and live for millennia.)

Differentiation

MCP-native, drop-in

A transparent proxy — point any MCP client at it, zero changes to agent or tool server.

Measured detection

A maintained attack corpus + eval harness producing real detection-rate and false-positive numbers, not marketing claims.

Behavioral session intelligence

Sequence-level analysis of what an agent is doing across a session vs. its declared role/goal.

Agent identity

Cryptographically bound identity per agent/session — "SPIFFE for agents."

The full loop

Everyone detects and blocks. Almost nobody does Reproduce → Repair → Verify — closing the loop is the moat.

Architecture

Olive is a transparent MCP proxy. To the agent's MCP client it looks like a normal MCP server; to the real tool server it looks like a normal MCP client. Everything that crosses it is inspected in both directions.

Two paths, one rule: the fast path is deterministic and enforces; the parallel path is intelligent and advises. LLM sentinels can only signal the circuit breaker / Commander, which apply deterministic containment.

Components implemented so far

Gateway core (src/olive/gateway/)

  • proxy.py — presents an MCP server over stdio (olive run) or streamable HTTP (olive serve); forwards initialize, tools/list, tools/call, resources/*, prompts/*; runs the inspector pipeline both directions.
  • context.py — SecurityContext: one frozen object per message (agent/session/role/tool/arguments-hash/direction/trust/resource-ref/task-resources).
  • pipeline.py — inspector pipeline; first non-allow verdict wins; fails closed on any inspector error.
  • session.py / breaker.py — per-session state + circuit breaker: trips to quarantine after repeated blocks or a sentinel signal; reversible human release; idle eviction (quarantine never evicted).
  • ratelimit.py — per-role sliding-window throttle, independent lock from the breaker.
  • upstreams.py — multi-upstream: one gateway fronts several tool servers, tools namespaced server.tool.
  • mode.py — OperatingMode (normal/suspicious/siege), core-owned value the breaker holds; tunes containment thresholds.
  • telemetry.py — async telemetry seam to the intelligence layer (default no-op).
  • approvals.py — capability-gated approval registry for hold decisions.

Inspectors (src/olive/inspectors/)

  • policy.py — outbound role-based tool allowlist, default-deny.
  • context_policy.py — outbound, refine-only: task-binding, classification ceilings, approval holds.
  • patterns.py — inbound deterministic injection-phrase matching (layer zero), Unicode-normalized.
  • decode.py — layer 0.5: decodes base64/hex/rot13/url-encoding + folds homoglyphs before re-running the trigger matcher.

Identity (src/olive/identity/)

  • tokens.py — Mock CA, RS256 JWT signing/verification (algorithm pinned, expiry + audience checked).
  • claims.py — IdentityClaims: role comes from verified identity, never self-asserted config.

Transport (src/olive/transport/)

  • http.py — streamable HTTP with bearer-token enforcement on the wire (401 before the gateway is reached); capability-gated admin endpoint for session release.

Store (src/olive/store/)

  • events.py — SQLite (aiosqlite): events, incidents, tool_baselines tables. Raw payloads never stored — hashes + bounded evidence excerpts only.

Intelligence layer (src/olive/intelligence/, advisory only)

  • sentinels.py — Prompt-Injection, Data-Leak, Behavior sentinels.
  • runner.py — SentinelRunner: aggregates signals, the only place a signal becomes breaker.trip.
  • reporter.py — structured, human-readable incident reports.
  • remediation.py — remediation cycle ledger: reproduced → fix-proposed → verified → approved → learned.
  • commander.py — Security Commander: deterministic, sole caller of breaker.set_mode.
  • bus.py — incident-object bus: signed, append-only, async pub/sub between departments.
  • departments.py — Defense + Remediation departments wired via build_runtime_org.

Evals & CLI

  • evals/run_evals.py — loads the 62-case attack corpus, runs the real inspector pipeline, reports detection rate / false-positive rate / latency / per-category breakdown; CI regression gate (evals/baseline.json).
  • cli.py — olive run / serve / reset-baselines / cycle (open/propose/verify/approve/learn/show).

Non-negotiable security rules

1. All tool output is untrusted input. Trust labels only decide how much inspection runs, never zero.
2. LLM verdicts are advisory only. Enforcement (allow/block/quarantine/mode) is made by deterministic code. An LLM may only emit a signal.
3. Never log raw payloads. SHA-256 hashes + bounded evidence excerpts (≤200 chars) only.
4. Fail closed. If an inspector or pipeline errors, the decision is block, logged with the error as evidence.
5. Every decision is auditable. Each allow/block/hold/quarantine writes an event row with the rule that fired and why.

The "company of agents" (runtime org)

The architectural law that governs all of it: agents provide intelligence; deterministic systems enforce authority. An LLM may conclude "this looks suspicious." Only deterministic code decides "this identity may not access payroll, ever."

1. Defense department built

SentinelRunner + sentinels (Prompt-Injection, Data-Leak, Behavior). Publishes detection objects to the bus when it acts.

2. Red-team / attack-simulation build-time

The .claude/agents/red-team agent generates bypass attempts against new/changed detection logic; every bypass becomes a corpus case.

3. Remediation / builder built

The builder agent proposes a fix as a reviewable diff; the remediation ledger walks it through reproduced → fix-proposed → verified → approved → learned, human-gated throughout.

4. Verification built

Not a new LLM — the qa agent + the real run_evals.py gate. "Verify" is only writable from an actual passing gate result.

5. Security Commander built

Deterministic; sole authority over the fleet's OperatingMode (normal/suspicious/siege); routes incident objects between departments.

6. Command & Coordination hierarchy future

Full Supervisor tier (Defense/Red-Team/Builder supervisors under the Commander) — deferred.

Operating modes

ModeBehavior
NormalStandard thresholds, normal permissions, scheduled simulations.
SuspiciousContainment threshold halves (quarantine sooner); more context rules escalate to hold.
SiegeThreshold collapses to one block; sensitive tools denied inline; new sessions default to hold.

Mode only ever escalates automatically (monotonic); de-escalation requires a capability-gated (olive:command) human.

What deliberately doesn't exist yet

  • The full Command & Coordination hierarchy (Commander → department supervisors → specialists).
  • Runtime Red-Team / Builder autonomy — only the build-time .claude/agents versions exist; nothing auto-attacks or auto-patches at runtime.
  • Auto-apply / auto-deploy of a proposed fix — permanently human-gated by design.
  • Credential/token freezing in Siege mode.
  • Cross-process / fleet-wide mode propagation; durable mode/bus across restarts (currently in-memory/per-process).
  • Cross-session / fleet behavioral baselines and the enterprise control plane.
  • A dashboard.

The "Later" bets

  • Real agent identity — "SPIFFE for agents", delegation chains, capability attenuation.
  • Enterprise control plane — centralized policy, agent inventory, attack replay, cross-session behavioral detection, compliance evidence, fleet management.
  • Business posture decision (open-core vs. closed) — deliberately deferred until after first external feedback.
